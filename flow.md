ฉันมีข้อมูล work flow ของ copy point (CP) เป็นหน่วยงานให้บริการเครื่องพิมพ์และกระดาษ ลักษณะการให้บริการมีหลายแบบขึ้นอยู่กับ requirement เช่น
- ให้บริการเช่าเครื่องพิมพ์พร้อมกระดาษ (แบบรายเดือน สัญญา 1,3,5 ปี/แบบคิดตามปริมาณการใช้งาน/แบบใช้งานเล็กน้อย คิดขั้นต่ำจากปริมาณการใช้งาน)
- ให้บริการเช่าเฉพาะเครื่องพิมพ์
- ทำสัญญากับพาร์ทเนอร์ โดยใช้เครื่องพิมพ์จากพาร์ทเนอร์ แต่เราให้ service
เป็นต้น
หากคุณเป็น process improvement จากขั้นตอนการทำงานเหล่านี้ช่วยวิเคาะห์ และนำเสนอวิธีการทำงานที่มีประสิทธิภาพยิ่งขึ้น แต่ไม่เปลี่ยนลักษณะการทำงานจนเกินไปให้หน่อย 
ขั้นตอนการทำงาน 
1. Sale visit ลูกค้า (มีโบชัวร์รายการสินค้าจาก MKT) -ลูกค้าติดต่อเข้ามาได้หลายช่องทาง เช่น webpage, call center, sale visit
2. Sale นำส่งข้อมูลให้ MKT เพื่อพิจารณาว่าควรขายแพ็กเกจใด ทาง LINE ส่วนตัว (pain : ค้นหาข้อมูลเก่าๆยาก)
ข้อมูลประกอบการพิจารณาแพ็กเกจ 
- A4/A3 
- Printer only / Multi-function 
- สี / ขาวดำ 
- ปริมาณการพิมพ์ต่อเดือน 
- Past record ลูกค้า (ยี่ห้อกระดาษ/เครื่อง) เพื่อเทียบเคียงสเป็คเสนอลูกค้า 
3. MKT ออกใบเสนอราคา เนื่องจากต้องพิจารณาราคา case by case ---> quotation 
4. MKT หาเครื่องที่เหมาะกับความต้องการลูกค้า (เครื่องใหม่/เครื่องมือสอง) โดยกรอกรายละเอียดบน Google sheet plan (แต่ละ row เปรียบเสมือน ticket งานที่ทุกฝ่ายจะติดตามสถานะในแนวคอลัมน์ถัดๆไป) หากบันทึกเสร็จ check true เพื่อให้ข้อมูล pivot ไปแสดงยังชีตคนที่รับผิดชอบ 
5. Sale ส่งใบ quotation ให้ลูกค้า และประสานงานกับลูกค้าจนกว่าจะได้ quotation ที่พอใจ
6. หากลูกค้าสนใจจะกรอกใบจองเครื่อง ---> ใบจองเครื่องพิมพ์ 
ข้อมูลใบจองเครื่อง 
- ชื่อลูกค้า
- ชื่อผู้ติดต่อหน้างาน (พนักงานบริษัท) 
- จำนวนคอมที่ต้องการเชื่อมกับเครื่องพิมพ์
7. Sale ถ่ายใบจองเครื่องส่งให้การตลาดผ่าน LINE
8. MKT ออกสัญญา + PO ตามแบบฟอร์มที่มี โดยกรอกข้อมูลในช่องที่ปรับเปลี่ยนตามข้อมูลลูกค้า (pain: ไม่มีช่องสำหรับหมายเหตุ หากลูกค้ารายใดมีข้อมูลเพิ่มเติม MKT ต้อง add manual ในฟอร์มเอง)
9. Planner ที่รับงานต่อรับทราบผ่านตาราง pivot และกด check true ในชีต plan เดียวกันกับการตลาด ในคอลัมน์ถัดไป
10. ช่างรับทราบผ่านตาราง pivot และกด check true ในชีต plan พร้อมทั้งระบุวันที่ที่คาดว่าเครื่องจะเสร็จพร้อมนำติดตั้ง 
11. Planner รับทราบแผนจากช่างและระบุแผนการนำส่งเครื่อง
- จัดส่งที่ใด (ต่างจังหวัดหรือกรุงเทพ)
- จัดส่งเมื่อไหร่ 
12. ช่างระบุผู้เข้าติดตั้ง และวันที่อัพเดตข้อมูล ---> ใบงานติดตั้งสำหรับช่าง 
ข้อมูลในใบงานติดตั้ง 
- ชื่อลูกค้า 
- ชื่อลูกค้า 2,3 (กรณีลูกค้า 1 รายสั่งให้จัดส่งหลายที่) ต้องการเก็บข้อมูลเหล่านี้เพื่อเป็นเป็นข้อมูลประกอบการเข้า service ในอนาคตได้ เช่น ลูกค้ารายนี้ มีเครื่องอยู่ที่ใดบ้าง เป็นต้น เพราะบางทีลูกค้าบอกให้ไป service แต่บอกแค่สถานที่รวม เช่นตึกA แต่ตึกA มีเครื่องติดตั้งอยู่หลายชั้น
- ข้อมูลเครื่องพิมพ์ที่จะติดตั้ง
13. ช่างติดตั้งเสร็จกลับมากรอกข้อมูล Printer QR
ข้อมูล Printer QR
- เลข code
- serial number
- meter mono
- meter color
